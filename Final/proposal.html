Summary

Conventional camera captures light intensity and color to form a picture. However, some information is ignored during this process, such like the distribution of light from different direction. With this information, we can change the depth of field without adjusting and retaking the photo. This project aims to collect and store the information while rendering and provide a GUI which can show the picture with different light field in real-time.

Problem Description

A light field camera stores additional information to support adjusting the depth of field after taking the photo. Based on project 3, we can add on this support by storing the additional information while rendering. This information is quite useful because we can reconstruct a picture with different depth of field by using it. The challenging part is how to represent the light field and how to use it to refocus. The main idea is to use a gird to instead pixel. By this way, we not only record the color and intensity of light, but also its direction. In this case, the sensor plane becomes a microlens array.

Suppose the Main lens plane is uv and the microlens plane is st, then the light function can be represented as L(u, v, s, t). We can calculate this function by using the grid. For refocusing, u, v will change, while we can still use this function to calculate the light reach the sensor plane.

Goals and Deliverables

Basic goals

We plan to simulate a light field camera so that we can adjust aperture and refocus after the scene has already been rendered. To accomplish this, we need to capture more light information than just the radiance at each pixel. Instead of 2D photograph, we have to store the 4D light field. Light field can be represented as a function L(u,v,s,t). Our goal is to render light field images that store this L function so that we can refocus after the scene is rendered. In addition, we plan to implement a GUI with which users can adjust aperture and refocus the rendered image and observe the effect in real time.

Some further thoughts

Since we need to capture richer information, light field images will cost much more space and time to render. Therefore, efficiency becomes a problem. We can implement our project on GPU or try some other ways to improve efficiency.
